# AI Proposal Generator — Project Skill

## 1. Назначение проекта

Flask-веб-приложение для быстрой генерации коммерческих предложений по AI-автоматизации бизнеса.

Пользователь заполняет форму (клиент, отрасль, проблема, решение, цена, сроки), выбирает тип КП и стиль — и получает готовый документ за 30 секунд. Генерация через OpenAI GPT-4o-mini. Если API-ключ не задан — fallback-режим по шаблону.

Результат: 8 структурированных разделов + полный текст. Экспорт в MD, TXT, PDF (структура), PDF (текст КП). История последних 5 генераций на главной странице.

## 2. Структура проекта

```
ai-proposal-generator/
├── app.py                  # Flask-приложение: маршруты, генерация, PDF, история
├── requirements.txt        # Зависимости: flask, openai, fpdf2, python-dotenv
├── .env.example            # Шаблон переменных окружения
├── .gitignore              # Исключает .env, venv/, outputs/*.pdf и др.
├── README.md               # Документация проекта для GitHub
├── templates/
│   ├── index.html          # Главная страница: hero, форма, шаблоны, история
│   └── result.html         # Страница результата: секции КП, кнопки экспорта
├── static/
│   └── style.css           # Все стили: тёмная тема, glassmorphism, адаптив
├── outputs/                # Экспортированные файлы (не публикуются в git)
│   └── .gitkeep
└── screenshots/            # Скриншоты для README
```

**Ключевые функции в `app.py`:**
- `generate_with_openai()` — генерация через GPT-4o-mini
- `generate_fallback()` — генерация по шаблону без API
- `generate_pdf()` — PDF со структурными разделами
- `generate_fulltext_pdf()` — PDF с полным текстом КП
- `save_outputs()` — сохранение MD, TXT, двух PDF, JSON-метаданных
- `get_history()` — чтение последних 5 генераций из `outputs/*.json`

## 3. Команды запуска

```bash
python3 -m venv venv
source venv/bin/activate        # macOS / Linux
# venv\Scripts\activate         # Windows

pip install -r requirements.txt
cp .env.example .env            # затем вставь OPENAI_API_KEY в .env
python app.py                   # → http://localhost:8080
```

> **Важно:** порт 5000 на macOS занят AirPlay — приложение запускается на **8080**.

## 4. Переменные окружения

Файл `.env` (создаётся из `.env.example`):

```
OPENAI_API_KEY=sk-...your-key...
```

**Fallback-режим:** если `OPENAI_API_KEY` не задан или равен `your_api_key_here`, приложение автоматически генерирует КП по встроенному шаблону. На странице результата появится синий баннер «Демо-режим». Функциональность полная — только текст менее персонализирован.

## 5. Как вносить изменения

### Внешний вид (стили)
→ `static/style.css`

Основные переменные в `:root {}` в начале файла: цвета, радиусы, шрифты. Glassmorphism-эффекты — классы `.glass`, `.form-card`, `.section-card`. Адаптив — медиазапросы в конце блоков.

### HTML-форма и страницы
→ `templates/index.html` — главная страница, форма, быстрые шаблоны, история  
→ `templates/result.html` — страница результата, секции КП, кнопки экспорта

**Внимание:** не запускай авто-форматтер (Prettier) на файлах в `templates/` — он ломает синтаксис Jinja2, разбивая `{% if x == 'y' %}` на несколько строк.

### Промпт генерации КП
→ `app.py`, функция `generate_with_openai()`, переменные `system_prompt` и `user_prompt`

Акценты для каждого типа КП — словарь `PROPOSAL_TYPE_FOCUS` в начале файла.

### Быстрые шаблоны (автозаполнение формы)
→ `templates/index.html`, объект `TEMPLATES` в блоке `<script>` внизу файла

Каждый шаблон — объект с полями: `client`, `industry`, `problem`, `solution`, `price`, `timeline`, `style`, `proposal_type`.

### Типы коммерческих предложений
→ `app.py`, словари `PROPOSAL_TYPES` и `PROPOSAL_TYPE_FOCUS`

Добавь новый ключ в оба словаря — он автоматически появится в `<select>` формы (рендерится через `{% for key, label in proposal_types.items() %}`).

### Экспорт PDF
→ `app.py`:
- `generate_pdf()` — PDF со структурными разделами (цвета, шрифты, отступы)
- `generate_fulltext_pdf()` — PDF с полным текстом КП (парсинг markdown по блокам)
- `save_outputs()` — здесь подключаются оба генератора и сохраняются метаданные

- [ ] Главная страница открывается без ошибок (`http://localhost:8080`)

- [ ] Генерация в демо-режиме (без API-ключа) — синий баннер, результат по шаблону
- [ ] Генерация через OpenAI API — зелёный баннер, персонализированный текст
- [ ] На странице результата отображаются все 8 разделов КП
- [ ] Кнопка «Скопировать» копирует текст в буфер
- [ ] Скачивание MD и TXT — файлы открываются корректно
- [ ] Скачивание PDF (структура) — разделы с заголовками, тёмный текст
- [ ] Скачивание PDF (текст КП) — связный текст, читаемое форматирование
- [ ] Файлы `.md`, `.txt`, `.pdf`, `_full.pdf`, `.json` появляются в `outputs/`
- [ ] История генераций на главной показывает последние записи с кнопками MD/TXT/PDF/PDF(КП)
- [ ] README содержит актуальные скриншоты и инструкцию запуска

## 8. Правила для Claude Code

При работе с этим проектом обязательно соблюдай:

- **Не переписывать с нуля** — проект рабочий, вносить точечные изменения
- **Сохранять Flask** — не мигрировать на FastAPI, Django и другие фреймворки
- **Без базы данных** — история хранится в JSON-файлах в `outputs/`, этого достаточно
- **Без авторизации** — проект локальный, логин не нужен
- **Без React/Vue/Next.js** — только нативный HTML + CSS + ванильный JS
- **Маленькие шаги** — одно изменение за раз, не смешивать рефакторинг с фичами
- **Не трогать Jinja2 в шаблонах авто-форматтером** — сломает синтаксис `{% %}`
- **После изменений — тест** — запустить сервер и проверить чек-лист из раздела 6
- **PDF-шрифт** — используется Arial Unicode (`/Library/Fonts/Arial Unicode.ttf`), поддерживает кириллицу; символ `₽` заменяется на `руб.` перед записью в PDF
